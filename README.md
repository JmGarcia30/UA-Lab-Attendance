## 📖 Overview
UA Laboratory Attendance is a highly secure, centralized web application designed to automate student attendance tracking across university laboratories. Built to solve the vulnerabilities of traditional manual logging, this system replaces easily forgeable sign-in sheets with a **Zero-Trust digital infrastructure**.

## 🛡️ Core Security Architecture
This project goes beyond standard authentication by implementing a **Defense-in-Depth** security model:

1. **Zero-Trust Cryptography (ECC):** Implements the **Elliptic Curve Digital Signature Algorithm (ECDSA P-256)** via the native Web Crypto API. Private keys remain securely locked on the student's device. Attendance payloads (Student ID, Room, Timestamp) are digitally signed to mathematically prove identity and prevent data tampering in transit.
2. **GPS Geofencing:** Restricts portal access to specific university campus coordinates (75-meter radius by default) to prevent remote access and proxy attendance.
3. **Time-Based Session PINs:** Dynamic, short-lived session PINs generated by instructors ensure that students are physically present inside the correct laboratory room at the correct time.
4. **Data Privacy:** Utilizes `bcrypt` hashing for all sensitive credentials, including administrative passwords and student device recovery PINs.

## 💻 Technology Stack
* **Frontend:** Next.js (React), Tailwind CSS, Lucide Icons, `idb-keyval` (IndexedDB for secure client storage)
* **Backend:** Next.js Server Actions, Node.js Crypto Module
* **Database:** Cloud-hosted MySQL (deployed via Aiven), managed with Prisma ORM

## 👥 Role-Based Features
* **Student Portal:** Secure device registration, geofence validation, and one-click cryptographically signed attendance logging.
* **Faculty Dashboard:** Session generation, PIN management, and real-time attendance monitoring.
* **Admin Control Panel:** Complete oversight of system health, staff account creation, schedule management, and cryptographic audit logs.
